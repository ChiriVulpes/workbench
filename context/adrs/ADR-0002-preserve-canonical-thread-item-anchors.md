# ADR-0002: Preserve Canonical Thread Item Anchors

Codex transcript repair may use generic compact snapshot items like `item-5` to recover content missing from app-server thread reads, but canonical streamed items such as `rs_*` reasoning entries and command executions own their timeline positions. Do not dedupe or merge different canonical reasoning ids just because their text matches or is prefix-compatible; that can move later reasoning into earlier chat history. Cleanup may remove duplicated content from generic snapshot items when a canonical item already owns that segment, but it must preserve reasoning array indexes by blanking duplicate generic segments rather than compacting arrays. Client live-stream pruning may replace browser-created placeholders with canonical server items, but it must not collapse unrelated canonical reasoning ids.
